## Computer [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Computer%22%20in%20Java%20programming%20in%20detail)

A **computer** is an electronic device that processes data according to a set of instructions. It takes input, processes it, stores it, and produces output. Modern computers can perform billions of operations per second and are used for everything from browsing the web to scientific simulations.

Core components include the **CPU** (processor), **memory** (RAM), **storage**, and **input/output devices** like keyboards and screens.

## Language [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Language%22%20in%20Java%20programming%20in%20detail)

A **language** is a structured system of communication — a set of symbols, words, and rules used to convey meaning. Human languages like English, Hindi, or French allow people to communicate with each other.

In the context of computing, a language is any formal system with a defined **syntax** (rules) and **semantics** (meaning) that can be used to give instructions or represent information.

## Programming Language [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Programming%20Language%22%20in%20Java%20programming%20in%20detail)

A **programming language** is a formal language used to write instructions that a computer can understand and execute. It's the bridge between human intent and machine action.

Examples include Python, Java, C++, and JavaScript. Every programming language has its own syntax and rules, but they all serve the same purpose — telling the computer what to do.

There are broadly two types:

- **Low-level languages** (e.g., Assembly, Machine Code) — closer to what the hardware understands, fast but hard to write.
- **High-level languages** (e.g., Python, Java) — closer to human language, easier to write and read.

> **In short:** A *computer* is the machine, a *language* is a system of communication, and a *programming language* is the special language we use to communicate with and instruct computers.

## Why Java? [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Why%20Java%3F%22%20in%20Java%20programming%20in%20detail)

Java emerged specifically to solve a real problem that C and C++ had: code written for one operating system wouldn't run on another without being rewritten.

### The Problem Java Solved [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22The%20Problem%20Java%20Solved%22%20in%20Java%20programming%20in%20detail)

In the early 1990s, if you wrote a C++ program on Windows, it produced machine code tied to that specific hardware and OS. To run the same program on a Mac or Unix machine, you had to recompile it — and often fix incompatibilities. For a world rapidly connecting via the internet, this was a major obstacle.

### Java's Big Idea: The JVM [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Java%27s%20Big%20Idea%3A%20The%20JVM%22%20in%20Java%20programming%20in%20detail)

Sun Microsystems introduced the **Java Virtual Machine (JVM)**. Instead of compiling your code directly to machine instructions, Java compiles to an intermediate format called "**bytecode**." The JVM then interprets that bytecode on whatever platform it's running on.

> **"Write once, run anywhere."**

### Why It Branched from C++ [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Why%20It%20Branched%20from%20C%2B%2B%22%20in%20Java%20programming%20in%20detail)

Java was deliberately designed to feel familiar to C++ developers — it borrowed the curly-brace syntax, classes, and object-oriented style. But it removed the dangerous parts: **no manual memory management, no pointers, no platform-specific headers**. This made it much safer and easier to write large, reliable applications.

### Why It Became So Dominant [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Why%20It%20Became%20So%20Dominant%22%20in%20Java%20programming%20in%20detail)

Three things made Java explode in popularity:

1. **The internet boom** — web servers needed to run on all kinds of hardware, and Java's portability was perfect
2. **Enterprise adoption** — large companies loved that Java programs were stable, predictable, and easy to maintain
3. **Android** — when Google built Android in 2007, they chose Java as the primary language, putting it in billions of phones

## History of Java [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22History%20of%20Java%22%20in%20Java%20programming%20in%20detail)

**The Origin (1991–1994)** — James Gosling and a small team at Sun Microsystems started with a vision of programming interactive TV set-top boxes and embedded devices. The language was first called *Oak*, but when they discovered that name was already trademarked, they renamed it *Java* — allegedly over a cup of coffee. The real pivot came when the internet boom arrived.

**The Public Era (1995–1998)** — Java's launch was electrifying. Netscape Navigator embedded a Java runtime, letting websites run Java "applets" directly in the browser. Java 2 in 1998 introduced the Collections framework and the Swing UI toolkit.

**Enterprise Dominance (2004–2007)** — Java 5 was a landmark release — generics, enums, and annotations transformed how developers wrote code. Google's decision to build Android on Java in 2007 was the real game-changer.

**The Oracle Era (2010–present)** — Oracle's acquisition of Sun was controversial. It led to a famous lawsuit against Google over Android's use of Java APIs. Despite the drama, Java continued to evolve — Java 8 (2014) introduced lambdas and streams, and from Java 9 onward, Oracle shifted to a 6-month release cycle.

> Today Java is one of the **top 3 most used programming languages** in the world.

## Java Editions [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Java%20Editions%22%20in%20Java%20programming%20in%20detail)

### J2SE — Java 2 Standard Edition [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22J2SE%20%E2%80%94%20Java%202%20Standard%20Edition%22%20in%20Java%20programming%20in%20detail)

The **core foundation**. Contains fundamental libraries, JVM, core APIs (collections, file I/O, networking, multithreading, Swing UI toolkit).

> **Target:** Desktop applications, standalone tools, general-purpose programs.

### J2EE — Java 2 Enterprise Edition [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22J2EE%20%E2%80%94%20Java%202%20Enterprise%20Edition%22%20in%20Java%20programming%20in%20detail)

Takes J2SE and adds tools for **large-scale, server-side, business applications**: Servlets, JSP, EJB, JMS, JNDI.

> **Target:** Web servers, enterprise applications, large distributed systems.

### J2ME — Java 2 Micro Edition [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22J2ME%20%E2%80%94%20Java%202%20Micro%20Edition%22%20in%20Java%20programming%20in%20detail)

Stripped Java down to run on devices with limited resources — mobile phones, pagers, PDAs. Introduced CLDC, CDC, and MIDP profiles.

> Before Android existed, J2ME was how you put games and apps on a mobile phone. Snake on your Nokia? Very likely J2ME.

> **Target:** Feature phones, PDAs, embedded devices, smartcards.

### How They Relate [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22How%20They%20Relate%22%20in%20Java%20programming%20in%20detail)

- **J2SE** = middle ground (general purpose)
- **J2EE** = expands upward (enterprise capabilities)
- **J2ME** = shrinks downward (small devices)

Over time: J2SE → **Java SE**, J2EE → **Jakarta EE**, J2ME → largely replaced by **Android**.

## Java Environment [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Java%20Environment%22%20in%20Java%20programming%20in%20detail)

### JDK — Java Development Kit [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22JDK%20%E2%80%94%20Java%20Development%20Kit%22%20in%20Java%20programming%20in%20detail)

The **full package** a developer installs. Contains the compiler (`javac`), debugger, documentation generator (`javadoc`), and development tools. **Includes the JRE**.

> Think of JDK as the complete workshop — all the tools to build from scratch.
> **Who needs it:** Developers writing Java code.

### JRE — Java Runtime Environment [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22JRE%20%E2%80%94%20Java%20Runtime%20Environment%22%20in%20Java%20programming%20in%20detail)

What an **end user** needs to run a Java program. Contains the JVM plus core Java class libraries. **No compiler or development tools**.

> Think of JRE as the engine that runs a finished car.
> **Who needs it:** Anyone who wants to run a Java application.

### JVM — Java Virtual Machine [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22JVM%20%E2%80%94%20Java%20Virtual%20Machine%22%20in%20Java%20programming%20in%20detail)

The **heart of it all**. Reads `.class` files (bytecode) and translates them into native machine instructions for whatever OS and hardware it's running on.

> The JVM makes "write once, run anywhere" real.

Also handles: memory management (garbage collection), security, and thread management.

### How They Work Together — Full Flow [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22How%20They%20Work%20Together%20%E2%80%94%20Full%20Flow%22%20in%20Java%20programming%20in%20detail)

1. You write `Hello.java` — source code in human-readable Java
2. The JDK's compiler (`javac`) compiles it into `Hello.class` — platform-independent bytecode
3. You run it using the `java` command, which invokes the JRE
4. The JRE loads the core libraries needed by your program
5. The JVM inside the JRE reads the bytecode and translates it into native machine instructions
6. Your program executes

### The Nesting Relationship [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22The%20Nesting%20Relationship%22%20in%20Java%20programming%20in%20detail)

```
JDK (outermost) — tools to BUILD programs
  └── JRE — libraries to RUN programs
        └── JVM (innermost) — pure execution engine
```

> Every JDK contains a JRE, and every JRE contains a JVM.

## Java Buzzwords (Features of Java) [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Java%20Buzzwords%20%28Features%20of%20Java%29%22%20in%20Java%20programming%20in%20detail)

### 1. Simple [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Simple%22%20in%20Java%20programming%20in%20detail)

Removed confusing parts of C++ — pointers, operator overloading, manual memory management — while keeping familiar curly-brace syntax.

### 2. Object-Oriented [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Object-Oriented%22%20in%20Java%20programming%20in%20detail)

Everything is an object (except primitive types). Follows **encapsulation, inheritance, polymorphism, and abstraction**.

### 3. Platform Independent [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Platform%20Independent%22%20in%20Java%20programming%20in%20detail)

"**Write Once, Run Anywhere.**" Source code compiles to bytecode, not machine-specific instructions. Runs on any device with a JVM.

### 4. Secured [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Secured%22%20in%20Java%20programming%20in%20detail)

No pointers (can't access memory directly). JVM runs programs in a **sandbox**. Multiple layers: class loader, bytecode verifier, security manager.

### 5. Robust [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Robust%22%20in%20Java%20programming%20in%20detail)

Strong type checking, automatic garbage collection, exception handling, no pointers — dramatically reduces program failures.

### 6. Architecture Neutral [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Architecture%20Neutral%22%20in%20Java%20programming%20in%20detail)

Bytecode not tied to any processor architecture. Data type sizes are strictly defined: `int` is always 32 bits, `long` is always 64 bits — everywhere.

### 7. Portable [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Portable%22%20in%20Java%20programming%20in%20detail)

Same `.class` file runs on any platform without recompiling. Standard library is consistent across platforms.

### 8. Interpreted [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Interpreted%22%20in%20Java%20programming%20in%20detail)

Two-step execution: compiler produces bytecode → JVM interprets at runtime. Modern JVMs include **JIT (Just-In-Time) compiler** for performance.

### 9. High Performance [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22High%20Performance%22%20in%20Java%20programming%20in%20detail)

**HotSpot JVM** and **JIT compilation** identify frequently executed code ("hot spots") and compile to native code, achieving performance close to C/C++.

### 10. Multithreaded [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Multithreaded%22%20in%20Java%20programming%20in%20detail)

Built-in support via `Thread` class and `Runnable` interface. Well-suited for concurrent applications (web servers, responsive UIs).

### 11. Distributed [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Distributed%22%20in%20Java%20programming%20in%20detail)

Rich standard library support for TCP/IP, HTTP, FTP, and **RMI** (Remote Method Invocation). Natural fit for internet-era distributed systems.

### 12. Dynamic [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Dynamic%22%20in%20Java%20programming%20in%20detail)

Classes loaded at runtime. Supports **reflection** — programs can inspect and manipulate their own structure. Enables frameworks like Spring and Hibernate.

## Differences Between C and Java [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Differences%20Between%20C%20and%20Java%22%20in%20Java%20programming%20in%20detail)

| Feature | C | Java |
| ------- | - | ---- |
| Type | Procedural / structured | Object-oriented |

| Memory management | Manual — `malloc()` and `free()` | Automatic — garbage collection |
| Pointers | Supports pointers directly | No explicit pointers |
| Security | Less secure — direct memory access | More secure — sandbox execution |
| Compilation | Compiled to native machine code | Compiled to bytecode, interpreted by JVM |
| Portability | Not portable — must recompile per platform | Highly portable — same bytecode runs everywhere |
| Exception handling | No built-in (`errno`, manual checks) | Built-in (`try-catch-finally`) |
| Inheritance | Not supported | Fully supported |

| Goto statement | Supported | Not supported (reserved but unused) |

## Editor vs IDE [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Editor%20vs%20IDE%22%20in%20Java%20programming%20in%20detail)

### What is an Editor? [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22What%20is%20an%20Editor%3F%22%20in%20Java%20programming%20in%20detail)

A **text editor** is a simple program for writing and editing plain text or code. No built-in compiler, debugger, or code suggestions. You write code, save it, then manually compile and run from the command line.

**Examples:** Notepad, Notepad++, EditPlus, Sublime Text, VS Code (basic mode)

### What is an IDE? [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22What%20is%20an%20IDE%3F%22%20in%20Java%20programming%20in%20detail)

An **IDE (Integrated Development Environment)** is a complete, all-in-one software package for writing, compiling, running, debugging, and managing code.

Key features:
- **Code completion** — suggests method/variable names as you type
- **Syntax highlighting** — colors keywords, strings, errors
- **Built-in compiler** — compile with a button click
- **Debugger** — pause execution, inspect variables, step through code
- **Error highlighting** — underlines mistakes in real time
- **Project management** — organizes files, folders, libraries
- **Refactoring tools** — rename across an entire project safely
- **Version control** — Git support built in

### Java Editors [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Java%20Editors%22%20in%20Java%20programming%20in%20detail)

### ASCII [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22ASCII%22%20in%20Java%20programming%20in%20detail)

**ASCII (American Standard Code for Information Interchange)** — character encoding standard using **7 bits** (128 characters).

### Unicode [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Unicode%22%20in%20Java%20programming%20in%20detail)

- Uses 16 bits or more (UTF-8, UTF-16, UTF-32)
- Supports 1+ million characters
- Includes all ASCII + international languages + emojis

**Java uses Unicode (UTF-16) internally.** Every `char` is **2 bytes (16 bits)**. ASCII is a subset of Unicode, so it works automatically.

### Input Using BufferedReader + InputStreamReader [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Input%20Using%20BufferedReader%20%2B%20InputStreamReader%22%20in%20Java%20programming%20in%20detail)

These two classes work together from the `java.io` package:

```
System.in → InputStreamReader → BufferedReader → Your Code
(bytes)     (converts to chars)   (buffers + readLine)
```

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;

class InputBuffISR {
    public static void main(String[] args) {
        InputStreamReader isr = new InputStreamReader(System.in);
        BufferedReader br = new BufferedReader(isr);

        String message = "";
        System.out.print("Enter the Message: ");
        try { message = br.readLine(); } catch (Exception e) {}
        System.out.println("Entered Message: " + message);

        // Reading numeric data
        int x = 0;
        System.out.print("Enter any int value: ");
        try { x = Integer.parseInt(br.readLine()); } catch (Exception e) {}
        System.out.println("Entered int value: " + x);

        // Reading double value
        double val = 0.0;
        System.out.print("Enter any double value: ");
        try { val = Double.parseDouble(br.readLine()); } catch (Exception e) {}
        System.out.println("Entered double value: " + val);
    }
}
```

**Key Points:**
- Always call `readLine()` — returns the whole line as String
- Parse manually using `Integer.parseInt()`, `Double.parseDouble()`, etc.
- Declare `throws IOException` or use `try-catch`
- Go-to for **competitive programming** due to speed

### Input Using Scanner Class [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Input%20Using%20Scanner%20Class%22%20in%20Java%20programming%20in%20detail)

The most **beginner-friendly** way. Part of `java.util`.

```
System.in → Scanner → Your Code
(bytes)     (parses tokens automatically)
```

#### Core Methods

| Method | Reads | Example Input |
| ------ | ----- | ------------- |
| `nextLine()` | Full line as String | "Hello World" |
| `next()` | Single word/token | "Hello" |
| `nextInt()` | Integer | 42 |
| `nextDouble()` | Double | 3.14 |

| `nextFloat()` | Float | 2.5 |

```java
import java.util.Scanner;

class InputUsingScanner {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter the Message: ");
        String message = sc.nextLine();
        System.out.println("Entered Message: " + message);

        System.out.print("Enter any int value: ");
        int x = sc.nextInt();
        System.out.println("Entered int value: " + x);

        System.out.print("Enter any double value: ");
        double val = sc.nextDouble();
        System.out.println("Entered double value: " + val);

        System.out.print("Enter any char value: ");
        sc.nextLine(); // flush buffer

    }
}
```

**Key Points:**
- Use `nextInt()`, `nextDouble()` etc. for direct parsing
- Always use `sc.nextLine()` to **flush the buffer** after `nextInt()` when switching to `nextLine()`
- Close the scanner when done: `sc.close()`

### Input Using DataInputStream [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Input%20Using%20DataInputStream%22%20in%20Java%20programming%20in%20detail)

`DataInputStream` reads **primitive data types** from an input stream in a machine-independent way. Part of `java.io`.

> **Not recommended** for keyboard input (`readLine()` is deprecated). Best used with `DataOutputStream` for **binary file I/O** and **network sockets**.

```java
import java.io.*;

public class DataStreamExample {
    public static void main(String[] args) throws IOException {
        // Write binary data
        DataOutputStream dos = new DataOutputStream(new FileOutputStream("data.txt"));
        dos.writeInt(101);
        dos.writeDouble(99.99);
        dos.writeUTF("Hello Java");
        dos.close();

        // Read binary data
        DataInputStream dis = new DataInputStream(new FileInputStream("data.txt"));
        int id = dis.readInt();
        double price = dis.readDouble();
        String message = dis.readUTF();
```

### Input Using StreamTokenizer [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Input%20Using%20StreamTokenizer%22%20in%20Java%20programming%20in%20detail)

Breaks input streams into **tokens** (words, numbers, symbols). Useful for parsing structured text.

```java
import java.io.*;

public class StreamTokenizerExample {
    public static void main(String[] args) throws IOException {
        String data = "Hello 123 World 45.6";
        Reader reader = new StringReader(data);
        StreamTokenizer st = new StreamTokenizer(reader);

        while (st.nextToken() != StreamTokenizer.TT_EOF) {
            if (st.ttype == StreamTokenizer.TT_WORD) {
                System.out.println("Word: " + st.sval);
            } else if (st.ttype == StreamTokenizer.TT_NUMBER) {
                System.out.println("Number: " + st.nval);
            } else {
                System.out.println("Symbol: " + (char) st.ttype);
            }
        }
    }
}
```

## Operators in Java [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Operators%20in%20Java%22%20in%20Java%20programming%20in%20detail)

### 1. Arithmetic Operators [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Arithmetic%20Operators%22%20in%20Java%20programming%20in%20detail)

| Operator | Description | Example |
| -------- | ----------- | ------- |
| `+` | Addition | `a + b` |
| `-` | Subtraction | `a - b` |
| `*` | Multiplication | `a * b` |
| `/` | Division | `a / b` |
| `%` | Modulus (remainder) | `a % b` |

```java
public class ArithmeticOperators {
    public static void main(String[] args) {
        int a = 10, b = 3;

        System.out.println("Addition       : " + (a + b));   // 13
        System.out.println("Subtraction    : " + (a - b));   // 7
        System.out.println("Multiplication : " + (a * b));   // 30
        System.out.println("Division       : " + (a / b));   // 3
        System.out.println("Division       : " + ((double)a / b)); // 3.333
        System.out.println("Modulus (a % b): " + (a % b));   // 1

        // Modulus with negative values
        System.out.println("Modulus(a % -b) : " + (a % -b));   // 1
        System.out.println("Modulus(-a % b) : " + (-a % b));   // -1
        System.out.println("Modulus(-a % -b): " + (-a % -b));  // -1

        // Modulus with fractional numbers
        System.out.println("Division       : " + (12.5 % b)); // 0.5
    }
}
```

**Key Points:**
- Integer division drops the decimal part: `10 / 3` gives `3`
- `+` also works for String concatenation
- Division by zero: integers → `ArithmeticException`; doubles → `Infinity`
- **Modulus sign rule:** sign of result = sign of **Numerator** (always)
- `%` works on both integers and floating-point
- `%` on floating-point division by zero returns `NaN`

### 5. Bitwise Operators [🤖 Ask ChatGPT](https://chatgpt.com/?q=Explain%20%22Bitwise%20Operators%22%20in%20Java%20programming%20in%20detail)

| Operator | Description | Example |
| -------- | ----------- | ------- |
| `&` | Bitwise AND | `a & b` |
| `\|` | Bitwise OR | `a \| b` |
| `^` | Bitwise XOR | `a ^ b` |
| `~` | Bitwise Complement | `~a` |
| `<<` | Left Shift | `a << 2` |
| `>>` | Right Shift | `a >> 2` |
| `>>>` | Unsigned Right Shift | `a >>> 2` |

#### Binary Conversion Shortcut

Weights: `128  64  32  16  8  4  2  1`

Example: `12` in binary → `8 + 4` → `0000 1100`

#### Bitwise Operations Example (a=10, b=12)

```
a & b:    0000 1010  AND  0000 1100  =  0000 1000 (8)
a | b:    0000 1010  OR   0000 1100  =  0000 1110 (14)
a ^ b:    0000 1010  XOR  0000 1100  =  0000 0110 (6)
```

#### Complement

```
~12 = -(12 + 1) = -13
```

#### Shift Operators

```
12 << 1 = 24    (multiply by 2)
12 << 2 = 48    (multiply by 4)
12 >> 1 = 6     (divide by 2)
12 >> 2 = 3     (divide by 4)
```

#### Unsigned Right Shift (`>>>`)

```
-1 >>> 28 = 15
11111111 11111111 11111111 11111111  (-1)
→ shift right 28, fill left with 0s
00000000 00000000 00000000 00001111  (15)
```

**Key Points:**
- Only work on **integer types** (`int`, `long`, `short`, `byte`, `char`)
- `<<` by 1 = multiply by 2; `>>` by 1 = divide by 2
- `>>>` treats number as raw bits without sign concern
- Widely used in cryptography, graphics, networking, performance-critical code
